2. The 3D Rendering Pipeline
Built with Three.js, the visualizer uses three distinct systems to represent data:

The Neural Core: An Icosahedron wireframe that represents the AI Model. Its scale and color temperature (HSL shift) are driven by the heat_index calculated in the backend.

The Energy Sink: A particle system with 3,000+ nodes. Particle velocity represents throughput, while color shifts from Cyan to Red to represent Grid Carbon Intensity.

Post-Processing: Uses emissive materials to simulate the "thermal glow" of high-density compute clusters.

🛣 Future Roadmap (Expanding the Research)
To reach a "State of the Art" (SOTA) visualization level, the following updates are planned:

[ ] Multi-Node Simulation: Add support for 1,000+ GPU clusters (e.g., GPT-4 scale) with specialized interconnect energy metrics.

[ ] Expanded Hardware Library: Include AMD Instinct MI300X, Google TPU v5p, and Apple M3 Max (Unified Memory) profiles.

[ ] Real-time Grid API: Integrate a live API (like Electricity Maps) to pull actual carbon intensity data based on the user's selected virtual location.

[ ] Inference vs. Training: A toggle to switch between the energy cost of generating a single sentence vs. training the entire model.
